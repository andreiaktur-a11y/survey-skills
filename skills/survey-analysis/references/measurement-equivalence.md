# Measurement Equivalence

The gate before you compare or pool groups (languages, modes, waves). If items don't measure the same construct the same way across groups, a "difference" may be an artifact of the instrument, not a real difference. This is the analysis-side companion to `../../survey-translation-adaptation/references/adaptation-vs-translation.md` and `../../../shared/3mc-considerations.md`.

## The three levels (plain terms)
- **Configural** — the same items load on the same construct in each group (the structure holds). Minimum to claim you're measuring the "same thing".
- **Metric** — the items relate to the construct with the same strength (scale intervals behave equivalently). Needed to compare **relationships/associations** across groups.
- **Scalar** — same thresholds/intercepts across groups. **Required to compare means / top-box levels across groups.** Without it, a higher score in one language group may just reflect different scale use, not higher satisfaction.

## Practical rule
- Comparing **levels** (which group is more satisfied): needs scalar equivalence. If untested or failed → **do not** rank groups on means; report each group separately with its own caveat.
- Comparing **drivers/associations** (what matters to each group): needs metric equivalence.
- **Pooling** groups into one estimate: only when equivalence holds and weights reflect the design; otherwise report disaggregated.

## What breaks equivalence
Translation that shifts meaning (fix upstream via TRAPD), culturally uneven social-desirability, different scale-use styles (acquiescence, extreme-response), mode effects (voice vs web). Detect via structural checks where n allows; where n is too small to test, **state the limitation and disaggregate** rather than assume equivalence.

## Default when in doubt
Report groups separately with per-group n and coverage. Silent pooling across untested language groups is a Major review finding (`../../survey-instrument-review/references/severity-rubric.md`).
