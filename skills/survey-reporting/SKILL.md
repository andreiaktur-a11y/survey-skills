# Survey Reporting

Communicate results so a reader can act on them and trust them.

## Load first
- `../../shared/tse-framework.md` — state the dominant remaining errors and their direction.
- `../../shared/3mc-considerations.md` — qualify group comparisons honestly.
- For the file itself, use the appropriate document skill (docx for reports, pptx for decks, xlsx for data tables/dashboards).

## Report structure — default template
```
# [Survey] Results — [period]
## Executive summary (what changed, what to do)
## Methodology (population, sample, mode(s), languages, response rate, weighting)
## Results by domain (top-box scores, distributions)
## Group comparisons (ONLY those that passed the equivalence gate; flag the rest)
## Key drivers
## Trend vs. prior wave / benchmark
## Limitations (dominant error sources, comparison-error caveats)
## Recommendations
```

## Principles
- Lead with the decision-relevant finding, not the methodology.
- Show uncertainty; don't present point estimates as exact.
- Any comparison that didn't pass the measurement-equivalence gate is reported as "not directly comparable," with the reason.
- Keep a methodology appendix sufficient for an auditor to follow.

## Output
The report or dashboard file (via the document skill), plus a one-paragraph executive summary suitable for an email.
