# Audit Checklist (review-specific)

> The substantive standard lives in `../../../shared/question-quality.md`. This file does NOT restate the rules — it turns them into an ordered audit pass and adds review-only concerns. For each item, run every section of `question-quality.md` as a pass/fail check; a violation is a finding, rated via `severity-rubric.md`.

## Pass order
1. **Objective gate** (`question-quality.md` §0) — does each item trace to a stated objective? Orphan items are findings.
2. **Question type** (§1) — is the type appropriate, and is its dominant error controlled (recall window, don't-know, etc.)?
3. **Wording** (§2) — one-idea, neutral, concrete, literacy-matched, bounded reference period, MECE options, translation-ready.
4. **Scales** (§3) — form, balance, labeling, midpoint, don't-know, within-construct consistency, data-type vs. intended analysis.
5. **Order & context** (§4) — priming, sensitive placement, routing.
6. **Sensitivity** (§5) — items sensitive in some target groups but not others.
7. **Reuse/adaptation** (§6) — were validated items used where available; is the ASQT/ADQ choice sound?
8. **Pretesting** (§7) — was each language pretested?

## Review-only additions (not in the shared file)
- [ ] **Construct coverage** — do the items, taken together, cover the intended constructs? List gaps.
- [ ] **Redundancy** — items measuring the same thing without purpose.
- [ ] **Standard mapping** — coverage against the applicable standard (hand specifics to `domain-healthcare-cma`).
- [ ] **Cross-version consistency** — for multilingual instruments, do all language versions match in structure/order/scales?
